Snapping Demo

				This demo shows how the plugins behavior with CSS scroll snapping.

				

				
				

					

						

						Browser  Support
					  

						

							This sample should work on Chrome 69+ (Canary, due to be released in September 2018), Firefox, Edge, IE 11, and Safari (iOS).
						

					

					

						

						Chrome
					  

						

							Chrome version 69 has the most up-to-date CSS snap implementation. It uses scroll-snap-align, while other browsers still use scroll-snap-coordinate.
						

					

					

						

						Safari/iOS
					  

						

							Safari supports CSS snapping with a couple of caveats (in addition to using the -webkit- prefix):
						

						
    
							
  • You must set -webkit-overflow-scrolling: touch;.
    • 
						

						
    
							
  • The elements being scrolled must have overflow: none; (the default setting).
    • 
						

					

					

						

						Edge
					  

						

							On Edge, dragging the scrollbar thumb doesn't cause snapping. Touching or clicking the pagination area in the scroll bar does.
						

						

							Edge requires the -ms- prefix for the CSS properties related to snap points.
						

					

					

						

						IE 11
					  

						

							On IE11, snapping is supported for touch only.
						

						

							IE11 requires the -ms- prefix for the CSS properties related to snap points.
						

					

					

						

						Touch Support
					  

						

							CSS snapping works with touch on recent Android and iOS phones and tablets. It is a great option for implementing scroll-based interfaces on narrow screens.
